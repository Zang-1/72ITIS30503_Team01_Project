4. CƠ SỞ DỮ LIỆU - DATABASE SCHEMA (Prisma / SQLite)
================================================================================

DATABASE PROVIDER: SQLite (file: ./dev.db)
ORM: Prisma Client JS

--- MODEL: User ---
  id       : Int      (PK, auto-increment)
  name     : String   (Họ tên)
  email    : String   (Unique - Email đăng nhập)
  password : String   (Mật khẩu - lưu plain text trong DB này)
  role     : String   (Default: "Customer" | "Admin")

--- MODEL: Category ---
  id       : Int        (PK, auto-increment)
  name     : String     (Tên danh mục: "Vợt Cầu Lông", "Tennis"...)
  slug     : String     (Unique - URL slug: "vot-cau-long")
  parentId : Int?       (FK tự tham chiếu -> id của Category cha)
  parent   : Category?  (Quan hệ cha - CategoryHierarchy)
  children : Category[] (Quan hệ con - CategoryHierarchy)
  products : Product[]  (Các sản phẩm thuộc danh mục)

  --> Hỗ trợ danh mục phân cấp 2 cấp (Parent -> Children)
  Ví dụ: "Cầu Lông" (parent) -> "Vợt Cầu Lông", "Giày Cầu Lông" (children)

--- MODEL: Tag ---
  id       : Int      (PK, auto-increment)
  name     : String   (Tên tag)
  slug     : String   (Unique)
  products : Product[] (Quan hệ many-to-many với Product)

--- MODEL: Product ---
  id            : Int         (PK, auto-increment)
  title         : String      (Tên sản phẩm)
  slug          : String      (Unique - URL slug)
  price         : Float       (Giá cơ bản)
  imageUrl      : String?     (Default: "/vot-cau-long-yonex.jpg")
  description   : String?     (Mô tả sản phẩm)
  specifications: String?     (Thông số kỹ thuật)
  categoryId    : Int         (FK -> Category.id)
  category      : Category    (Quan hệ nhiều-một)
  tags          : Tag[]       (Quan hệ nhiều-nhiều)
  orderItems    : OrderItem[] (Các mục đơn hàng liên quan)
  -- Trường bổ sung (từ nhánh develop) --
  image         : String?     (Ảnh URL thay thế)
  imageAlt      : String?     (Alt text ảnh)
  sku           : String?     (Unique - Mã SKU sản phẩm)
  regularPrice  : Float?      (Giá gốc)
  salePrice     : Float?      (Giá khuyến mãi)
  stockStatus   : String      (Default: "In stock")
  stockQty      : Int         (Default: 0 - Số lượng tồn kho)
  attributes    : Attribute[] (Các thuộc tính sản phẩm)
  variations    : Variation[] (Các biến thể sản phẩm)

--- MODEL: Order ---
  id            : String    (PK - Tự tạo theo format "ORD0001")
  customerName  : String    (Tên khách hàng)
  email         : String
  phone         : String
  address       : String
  paymentMethod : String    ("COD" hoặc "Bank Transfer")
  status        : String    (Default: "Pending" | "Processing" | "Completed")
  total         : Float     (Tổng tiền đơn hàng)
  createdAt     : DateTime  (Default: now())
  items         : OrderItem[]

--- MODEL: OrderItem ---
  id          : Int     (PK, auto-increment)
  orderId     : String  (FK -> Order.id, onDelete: Cascade)
  productId   : Int     (FK -> Product.id)
  variationId : Int?    (ID biến thể tùy chọn: 3 hoặc 4 cho 3U/4U)
  quantity    : Int
  price       : Float   (Giá tại thời điểm đặt hàng)

--- MODEL: Attribute ---
  id        : Int         (PK, auto-increment)
  name      : String      (Tên thuộc tính: "Trọng lượng", "Độ cứng"...)
  productId : Int         (FK -> Product.id)
  product   : Product
  variations: Variation[] (Các biến thể thuộc attribute này)

--- MODEL: Variation ---
  id           : Int       (PK, auto-increment)
  value        : String    (Giá trị biến thể: "3U", "4U", "S", "M"...)
  sku          : String    (Unique - Mã SKU biến thể)
  regularPrice : Float?    (Giá gốc biến thể)
  salePrice    : Float?    (Giá khuyến mãi biến thể)
  stockStatus  : String    (Default: "In stock")
  stockQty     : Int       (Default: 0)
  productId    : Int       (FK -> Product.id)
  attributeId  : Int       (FK -> Attribute.id)

5. KIẾN TRÚC & LUỒNG DỮ LIỆU
================================================================================

--- KIẾN TRÚC TỔNG QUAN ---
  Client (Browser)
      |
      v
  Next.js App Router
      |-- Server Components (fetch data trực tiếp từ Prisma)
      |-- Client Components (state, tương tác người dùng)
      |-- Server Actions ('use server' - xử lý form/mutation)
      |-- Edge API Routes (runtime: 'edge' - siêu nhanh)
      |
      v
  Prisma ORM
      |
      v
  SQLite Database (dev.db)

--- RENDERING STRATEGY ---
  - Trang chủ (/)         : Server Component (SSR - render server, fetch Prisma)
  - Trang admin (/admin)  : Client Component (CSR - dùng Server Action)
  - Trang sản phẩm       : Server Component + Client sub-components
  - CartDrawer            : Client Component (state phức tạp)
  - CartContext           : Client Context (localStorage persistence)

--- LUỒNG MUA HÀNG ---
  1. User duyệt sản phẩm trên trang chủ hoặc trang danh mục
  2. Click "Thêm vào giỏ" -> addToCart() được gọi trong CartContext
  3. CartDrawer tự động mở ra (isOpen = true)
  4. User xem giỏ hàng, điều chỉnh số lượng, xóa sản phẩm
  5. Click "Tiến hành đặt hàng" -> step chuyển sang 'checkout'
  6. User điền form: Tên, Email, SĐT, Địa chỉ, Phương thức thanh toán
  7. Validation phía client (regex phone VN, format email...)
  8. Submit form -> gọi checkoutAction() (Server Action)
  9. Server: tính tổng tiền, tạo OrderID (ORD0001...), lưu Order + OrderItems
  10. Trả về kết quả, hiển thị màn hình thành công với mã đơn hàng
  11. Cart được xóa sạch (clearCart)

      - Tên TK: GROUP 1

- STEP 3 - Success: Hiển thị mã đơn hàng, tổng tiền, trạng thái "Chờ xác nhận"

- Shipping fee logic: Đơn < 5.000.000đ phí 35.000đ, từ 5tr miễn phí

FILE: webapp/context/CartContext.tsx
------------------------------------------------------------------------
Loại: Client Context
Export: CartProvider, useCart
Interface CartItem:
  { id, title, price, imageUrl, quantity, variationId? }

Tính năng:
- Persistence: Lưu/tải giỏ hàng từ localStorage key 'shopping-cart'
- Hydration-safe: Dùng isHydrated state tránh SSR mismatch
- addToCart(item, quantity=1): Nếu đã có (cùng id + variationId) thì tăng qty
- removeFromCart(id, variationId?)
- updateQuantity(id, quantity, variationId?): Nếu qty <= 0 thì xóa
- clearCart(): Xóa toàn bộ giỏ
- Computed values:
    * totalItems: Tổng số lượng sản phẩm
    * subtotal: Tổng tiền chưa ship
    * shippingFee: 35.000đ nếu subtotal < 5.000.000đ, ngược lại = 0
    * grandTotal: subtotal + shippingFee
    * totalPrice: = subtotal (alias để tương thích ngược)

------------------------------------------------------------------------
FILE: webapp/app/actions/checkout.ts (Server Action)
------------------------------------------------------------------------
Directive: 'use server'
Export: checkoutAction(data: CheckoutData)

Luồng xử lý:
  1. Validate: Kiểm tra đầy đủ các trường, items không rỗng
  2. Tính tiền: subtotal, shippingFee (35k nếu < 5tr), totalAmount
  3. Tạo OrderID: Đếm số đơn hiện tại, tìm ID kế tiếp dạng "ORD0001"
     (thử tối đa 20 lần nếu trùng, fallback: ORD{timestamp})
  4. Tạo Order: prisma.order.create() với nested items.create
  5. Return: { success, orderId, totalAmount }

12. GHI CHÚ KỸ THUẬT QUAN TRỌNG
================================================================================

  1. Next.js 16.x có breaking changes so với Next.js 13/14/15
     -> Đọc node_modules/next/dist/docs/ trước khi code (theo AGENTS.md)

  2. TailwindCSS v4 thay đổi cú pháp so với v3:
     -> Dùng @tailwindcss/postcss thay vì tailwindcss plugin cũ
     -> Không có tailwind.config.js (v4 auto-detect)

  3. Prisma với SQLite: Không hỗ trợ Edge Runtime
     -> API route /api/specifications dùng Edge Runtime nên KHÔNG thể dùng Prisma
     -> Trang admin và checkout dùng Server Components/Actions (Node.js runtime)

  4. CartDrawer xử lý cả view cart + checkout trong 1 component
     -> Trang /checkout là separate page dùng cho coupon code feature

  5. variationId trong CartItem là number (3 hoặc 4) tương ứng 3U/4U
     -> Không phải ID từ DB Variation model

  6. Mật khẩu trong DB lưu plain text (không hash)
     -> Chỉ phù hợp cho môi trường học tập, KHÔNG dùng production

  7. Không có authentication middleware
     -> Bất kỳ ai cũng có thể truy cập /admin

  8. OrderID được tạo tuần tự (ORD0001...) không phải UUID
     -> Có race condition tiềm ẩn trong môi trường concurrent
